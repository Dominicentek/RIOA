RIOA is a functional interpreted programming language created for Truttle1's practical language jam. It's an acronym for (R)un (I)f (O)r (A)nd, which is the main feature of this language. To get started, download the interpreter from releases page on the GitHub reposity. The interpreter requires JRE 8 to be installed on your computer. More detailed instructions on how to use the interpreter can be found in the interpreter page of this documentation.
Defining variables is simple. This is how you define a variable variable = 0 Variables are not just numbers. There are also strings, booleans and arrays. string = "this is a string"
boolean = true
array = ["string", 0.5, boolean, false] There are also operator assignments: x = 5
x += 2
 Which is an equivalent to: x = 5
x = x + 2
 If you want to increment/decrement a number by just 1, you can do x++ x--
If you want to get a specific element in an array or specific character in a string, you can use the array accessor array = [1, 2, 3]
firstElement = array[0]

string = "123"
firstCharacter = string[0]
 You can also get length of the string or array by not specifying any value at all in the array accessor length = array[]
There are several different operators. All of these can be used in an assignment. Those include:
+ Adds two numbers together or if any of the values are strings, the values get concated into the string
- Subtracts two numbers together
* Multiplies two numbers together
/ Divides two numbers together
% Modules two numbers together
** Powers two numbers together
>> Bitshift to right
<< Bitshift to left
& Bitwise AND
| Bitwise OR
^ Bitwise XOR
&& Conditional AND
|| Conditional OR
^^ Conditional XOR
== Conditional equals
!= Conditional not equals
< Less than
> Greater than
<= Less than or equal to
>= Greater than or equal to

There are also modifiers. These are like operators but they work with 1 value. It is a singular symbol put behind a value, like this:

x = true
x = !x Here's a list of them:
- Negates the value
~ Flips all the bits in a number
@ Reverses the bits in a number
! Conditional inverse
Here's how an if statement looks like: if condition {} It doesn't use any brackets, though you can use them if you want to. This code simply executes code in the curly braces if the condition is true
This is the main feature of the language. Say you have a case like this: if condition1 {}
if condition2 {}
if condition3 {}
if condition1 && condition2 && condition3 {} We can replace that with: if condition1 {}
and if condition2 {}
and if condition3 {}
run {} That just looks way cleaner. You can also do this: if condition1 {}
and if condition2 {}
or if condition3 {}
run {} Which is an equivalent to: if condition1 {}
if condition2 {}
if condition3 {}
if condition1 && condition2 || condition3 {}
While loops are like if statements, but they execute code over and over again until the condition is not met. while condition {} You can use 2 keywords to manage the loop. continue and break. continue simply stops the current loop cycle and immediately turns to the next loop cycle. break simply breaks out of the loop completely. You can also break out of multiple loops like this: while condition1 {
while condition2 {
break 2
}
}
Errors happen if you wrote bad code or when there's just something wrong. Here are all the types of them:

SyntaxError - Mostly thrown for writing bad code
DefineError - Happens when trying to access or modify undefined variable, calling undefined functions or creating duplicate functions
RangeError - Trying to access element of an array or string that is out of bounds
TypeError - Using incorrect type for an operation, or when a function argument requires specific type(s)
IOError - Mostly thrown for errors in IO operations
GraphicsError - Thrown when graphics API is not initialized yet
InternalError - These are errors that occured in the interpreter itself and not in the code.

The interpreter immediately exits out of the program to prevent any issues from happening. But how do we stop the interpreter from closing and want to handle the error for ourselves? We use try-catch statement. try {}
catch errorName errorMessage lineNumber columnNumber {}
 Whenever there's an error in the try block, it skips to the catch block and handles the error there. There are 4 variables that get created in the catch block that can be defined when defining the catch block.

errorName - Name of the error
errorMessage - The error message
lineNumber - Line number on where the error occured
columnNumber - Column number on where the error occured

If there's no error in the try block, the catch block is skipped and never executed.
Functions are code blocks that are not initially executed, but can be executed instantly from somewhere else. All the code happens in there. You can define other functions or import libraries to the code. You can also pass arguments into these functions. You can define a function like this: func function() {} To define arguments, you can do this: func function(argument1, argument2) {} Now the function has to be called with 2 arguments or it will throw an error. You can also limit the type of the argument like this: func function(number numberArgument, string array stringArrayArgument) {} Argument 1 now has to be a number, and argument 2 has to be a string or an array
Every program has to have a main function which takes 1 array argument. This argument contains all the CLI arguments passed from the command line func main(args) {} To call a function, you do this: function(argument1, argument2)
Comments are simple. They are the way to add text to your code without actually contributing to the code itself. // This is a single line comment!
/*
This is a multi
line comment!
*/
You can import global variables and functions from a separate source file. These are called "libraries". This is how you import them into your source file. import "library.rioa" These have to be specified at the very top of your source file.
The interpreter provides 88 built-in functions that are automatically imported. These functions do all sorts of things like graphics, input, math and IO. Here's a list of them:

print(value) - This function simply prints the value to the console
println(value) - Exactly the same, but adds a new line after the message
input() - Asks the user for input and returns it
exit(number exitcode) - Exits the program with exit code>
wait(number duration) - Waits a specific amount of milliseconds
waitnano(number duration) - Waits some amount of nanoseconds, can be offsetted by a few hundred depending on processor speed
typeof(value) - Returns the type of value, as string
parsenum(string value) - Converts string to a number
includes(string string, string regex) - Checks if the string contains the regex
contains(array array, content) - Checks if the array contains the value
split(string string, string regex) - Splits the string into a string array based on regex
join(array stringarr, string delimiter) - Joins a string array by delimiter
replace(string string, string replacement) - Replaces regex by a replacement
sub(string array value, number beginindex, number endindex) - Returns substring/subarray based on begin index and end index
bytestostring(array bytearray) - Converts a byte array to string
stringtobytes(string string) - Converts a string to byte array
tocharcode(string character) - Takes a string with exactly 1 character and converts it to an ASCII value
fromcharcode(number charcode) - Converts ASCII char code to a character
lowercase(string string) - Converts every string's character into a lowercase version of itself
uppercase(string string) - Converts every string's character into an uppercase version of itself
gfxinit(string title, number width, number height) - Initializes graphics API and opens a window
gfxcolor(number r, number g, number b, number a) - Sets a color from the graphics API
gfxrectfill(number x, number y, number width, number height) - Draws a filled rectangle on the screen
gfxrectdraw(number x, number y, number width, number height) - Draws an outline of a rectangle on the screen
gfxcirclefill(number x, number y, number radius) - Draws a filled circle on the screen
gfxcircledraw(number x, number y, number radius) - Draws an outline of a circle on the screen
gfxfont(string fontname, number fontsize) - Sets a font for the graphics API
gfxstring(number x, number y, string text) - Draws text on the screen using the font used
gfximagedynamic(number x, number y, string array data) - Draws an image with dynamic width and height on the screen. The array can be used as an ARGB matrix of pixels. The first element is the image width.
gfximagefixed(number x, number y, number width, number height, string array data) - Draws an image with fixed width and height
gfxline(number x1, number y1, number x2, number y2) - Draws a line
keypressed(number keycode) - Checks if a key is pressed
mouseposx() - Returns X position of the mouse on the window
mouseposy() - Returns Y position of the mouse on the window
mouseleftpressed() - Checks if the left mouse button is pressed
mouserightpressed() - Checks if the right mouse button is pressed
mousemiddlepressed() - Checks if the middle mouse button is pressed
mousescrollamount() - Returns the amount scrolled on the scroll wheel since the last check
closegfx() - Closes the window
audio(string filename) - Plays an audio file
ioread(string filename) - Reads a file and returns the bytes contained in the file
iowrite(string filename, array data) - Writes bytes to a file
iofolder(string filename) - Creates a folder
iodelete(string filename) - Deletes a file or a folder
iolistdir(string filename) - Lists all files in a directory, throws an IO error if a file

The interpreter is simple to use. Here's text that gets outputted by the interpreter when invalid CLI arguments are given:
Usage:
java -jar RIOA.jar [options] sourcefile [args...]

-output : Outputs the returning value of main function
-highlight : Outputs syntax highlighted ANSI formatted text file with source code
-run : Forcefully runs the code (used with -highlight to run both actions)
